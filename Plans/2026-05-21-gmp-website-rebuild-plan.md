# GMP Website Rebuild Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the Growth Mindset Parenting website as a production-quality Next.js static site that is mobile-first, pixel-perfect to the V6 Workshop design, and best-in-class for SEO and AI/LLM search.

**Architecture:** Next.js 14 App Router with `output: 'export'` (pure static HTML shipped to Vercel). All pages pre-rendered at build time. Content stored as plain JS data files — add a letter by adding one object. No database, no CMS.

**Tech Stack:** Next.js 14, React 18, plain CSS (ported from design), Vercel static hosting.

```jsx
'use client';
import { useState } from 'react';

// Set these in Vercel environment variables before launch.
// KIT_FORM_ID: found in Kit dashboard → Forms → your form → click the form → the number in the URL
// KIT_API_KEY: Kit dashboard → Settings → Developer → API Key (the public key, safe to expose)
const KIT_FORM_ID = process.env.NEXT_PUBLIC_KIT_FORM_ID;
const KIT_API_KEY = process.env.NEXT_PUBLIC_KIT_API_KEY;

export default function SubscribeForm({ inputPlaceholder = 'you@yourkitchen.com', buttonLabel = 'Subscribe →' }) {
  const [email, setEmail] = useState('');
  const [status, setStatus] = useState('idle'); // idle | submitting | success | error

  async function handleSubmit(e) {
    e.preventDefault();
    setStatus('submitting');
    try {
      const res = await fetch(
        `https://api.convertkit.com/v3/forms/${KIT_FORM_ID}/subscribe`,
        {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({ api_key: KIT_API_KEY, email }),
        }
      );
      if (res.ok) {
        setStatus('success');
        setEmail('');
      } else {
        setStatus('error');
      }
    } catch {
      setStatus('error');
    }
  }

  if (status === 'success') {
    return (
      <p style={{ fontFamily: 'var(--sans)', fontWeight: 600, color: 'var(--paper)' }}>
        You&rsquo;re in. See you Sunday.
      </p>
    );
  }

  if (status === 'error') {
    return (
      <p style={{ fontFamily: 'var(--sans)', fontWeight: 600, color: 'var(--paper)' }}>
        Something went wrong. Try emailing Sean directly — reply to any Sunday letter.
      </p>
    );
  }

  return (
    <form className="v6-sub-form" onSubmit={handleSubmit}>
      <input
        type="email"
        required
        placeholder={inputPlaceholder}
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        aria-label="Email address"
        disabled={status === 'submitting'}
      />
      <button type="submit" disabled={status === 'submitting'}>
        {status === 'submitting' ? 'Subscribing…' : buttonLabel}
      </button>
    </form>
  );
}
```

  return LETTERS.find((l) => l.slug === slug) || null;
}

export function getAllSlugs() {
  return LETTERS.map((l) => l.slug);
}

export function getLettersPage(page = 1) {
  const all = getAllLetters();
  const totalPages = Math.ceil(all.length / LETTERS_PER_PAGE);
  const start = (page - 1) * LETTERS_PER_PAGE;
  const items = all.slice(start, start + LETTERS_PER_PAGE);
  return { items, page, totalPages, total: all.length };
}

  return getAllLetters()
    .filter((l) => l.featured)
    .slice(0, limit);
}

**Content writing guidelines for real letters (not placeholder):**
When Sean replaces placeholder content with real letters, follow these patterns — they are backed by the Princeton GEO paper and the citation absorption research, and improve LLM visibility by 30–60%:

- **Length:** Aim for 1,500–2,000+ words per essay. Research shows high-influence pages average ~1,900 words vs ~170 for low-influence (11x difference).
- **Section headings:** Use multiple H2s and H3s per letter. High-influence pages have 12x more headings than low-influence pages. Each heading should name the specific claim, not a vague topic.
- **Statistics:** Include specific numbers — ages, percentages, study findings. Example: "Research by Dr. David Yeager found that students told their intelligence was fixed scored 30% lower on retest tasks." Numbers improve AI citation by ~40–62%.
- **Definitions:** Define key terms explicitly in the body. "Growth mindset — the belief that intelligence and character can be developed through effort" improves absorption by ~57%.
- **Quotes:** Include verbatim quotes from Yeager, Siegel, or Damour with attribution. Quotation addition improves AI visibility by ~41%.
- **Cited sources:** Link to or name the research being referenced. Cite Sources improved AI visibility by ~30%.
- **How-to structure:** Numbered steps or clear action instructions. How-to content improves absorption by ~41%.
- **Avoid:** Q&A format as the primary structure — it actually decreases absorption by ~6% on its own. Use Q&A sparingly inside longer evidence-rich sections.

import { LETTERS } from '../content/letters.js';
import { SITE } from '../data/site.js';
import { writeFileSync } from 'fs';
import { join, dirname } from 'path';
import { fileURLToPath } from 'url';

const __dirname = dirname(fileURLToPath(import.meta.url));

const sorted = [...LETTERS].sort((a, b) => new Date(b.date) - new Date(a.date));

const items = sorted.map(l => `
  <item>
    <title><![CDATA[${l.title}]]></title>
    <link>${SITE.url}/letters/${l.slug}/</link>
    <guid isPermaLink="true">${SITE.url}/letters/${l.slug}/</guid>
    <pubDate>${new Date(l.date).toUTCString()}</pubDate>
    <description><![CDATA[${l.excerpt || l.dek || ''}]]></description>
  </item>`).join('');

const feed = `<?xml version="1.0" encoding="UTF-8"?>
<rss version="2.0" xmlns:atom="http://www.w3.org/2005/Atom">
  <channel>
    <title>${SITE.name}</title>
    <link>${SITE.url}</link>
    <description>${SITE.description}</description>
    <language>en-us</language>
    <atom:link href="${SITE.url}/feed.xml" rel="self" type="application/rss+xml" />
    ${items}
  </channel>
</rss>`;

writeFileSync(join(__dirname, '../out/feed.xml'), feed);
console.log(`RSS feed generated: out/feed.xml (${sorted.length} items)`);

## Task 16: Post-Deploy — Google Analytics + Search Console

This task runs AFTER the site is live. All steps are in a browser, no code changes.

- [ ] **Step 1: Set up Google Analytics 4**

1. Go to `analytics.google.com`
2. Click Admin (gear icon, bottom left) → Create → Property
3. Property name: `Growth Mindset Parenting`
4. Time zone: Central Time (Chicago)
5. Currency: USD → Next
6. Business details: Education / Small → Next
7. Choose "Web" as the platform
8. Website URL: `growthmindsetparenting.com`
9. Stream name: `GMP Website`
10. Copy the Measurement ID (format: `G-XXXXXXXXXX`)
11. Add this ID to Vercel env vars as `NEXT_PUBLIC_GA_ID` → trigger a redeploy
12. Back in GA4: verify data is flowing → Realtime report → open the live site in a separate tab → you should see 1 active user

- [ ] **Step 2: Set up Google Search Console**

1. Go to `search.google.com/search-console`
2. Add property → choose "Domain" → enter `growthmindsetparenting.com`
3. Google will show a DNS TXT verification record
4. Add that TXT record in your domain registrar's DNS settings
5. Click "Verify" in Search Console
6. Once verified: Sitemaps → Add sitemap → enter `sitemap.xml` → Submit
7. Expected: sitemap shows all pages in the index queue within 24–48 hrs

- [ ] **Step 3: Add Search Console verification code to site** (alternative method)

If DNS verification is blocked or takes too long, use the HTML tag method instead:
1. Search Console → Settings → Ownership verification → HTML tag
2. Copy the `content=""` value from the meta tag they give you
3. In `app/layout.jsx`, replace `PASTE_GOOGLE_SEARCH_CONSOLE_CODE_HERE` with that value
4. Redeploy → click Verify in Search Console

- [ ] **Step 4: Link GA4 ↔ Search Console**

1. In GA4: Admin → Property Settings → Search Console Links → Link
2. Select your Search Console property → Next → Save
3. This unlocks the "Search Console" reports inside GA4 — shows which Google queries drive traffic

- [ ] **Step 5: Confirm everything is wired**

In GA4 → Reports → Realtime: open the live site, confirm active users appear.
In Search Console → Overview: confirm property is verified, sitemap submitted.
